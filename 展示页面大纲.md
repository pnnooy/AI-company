# 桌面学术助手机器人 - 展示页面大纲

> **目标**: 用 HTML 网页代替传统剪辑视频，实现项目介绍/功能展示/视频嵌入的一体化展示页
> **风格参考**: `D:\AI_company\前端风格参考\VQA_page` (teal 主色调、Merriweather+Inter+Nunito+Comfortaa 字体体系)
> **状态**: 大纲阶段，不涉及代码实现

---

## 一、方案可行性分析

### ✅ 总体结论：完全可行，且比传统视频剪辑有显著优势

| 维度 | 传统视频剪辑 | HTML 展示页方案 |
|------|-------------|----------------|
| 制作难度 | 需要 Premiere/剪映 等多轨道编辑 | 只需拍摄独立短视频片段 + HTML/CSS 排版 |
| 修改迭代 | 改一处需重新渲染导出 | 改一处刷新浏览器即可 |
| 信息密度 | 线性播放，观众只能被动跟随 | 观众可主动导航，信息可分层呈现 |
| 互动性 | 无 | 可嵌入交互式 demo（如表情切换器） |
| 分发方式 | 视频文件/链接 | 一个 URL 即可（部署到 GitHub Pages） |
| 视频同步 | 需要精确对轨 | 两个独立 `<video>` 并列播放，各自控制 |

### ⚠️ 需要注意的关键问题

1. **视频自动播放限制**: 现代浏览器禁止自动播放有声视频。所有嵌入视频需设为 `muted` + `playsinline`，或由用户点击触发播放。
2. **视频文件大小**: 建议每个视频片段 5-15 秒、720p/1080p、H.264 编码，单个文件控制在 5MB 以内，保证加载速度。
3. **左右同步**: "一边聊天窗口视频 + 一边实物视频" 的并列展示，两个视频独立播放即可，不需要精确帧同步（观众自己会看两边）。

## 七、技术实现要点（供后续编码参考）

1. **视频 `<video>` 标签属性**:
   ```html
   <video muted loop playsinline preload="metadata"
          poster="thumbnail.jpg">
     <source src="video.mp4" type="video/mp4">
   </video>
   ```
   - `muted` → 允许自动播放
   - `playsinline` → iOS 不强制全屏
   - `loop` → 表情动画循环播放
   - `preload="metadata"` → 只预加载元数据，节省带宽

2. **第 3 页 JS 表情引擎**: 使用 `requestAnimationFrame` + 帧索引循环（每 200ms 切换一帧），8 个表情按钮切换当前动画集。

3. **响应式布局**: 使用 CSS Grid + `@media` 查询，确保在手机/平板/桌面端都有良好体验。

4. **性能优化**:
   - 图片使用 WebP 格式（减小 30-50%）
   - 视频延迟加载（Intersection Observer 触发）
   - 字体使用 `font-display: swap`

5. **部署**: 推荐 GitHub Pages（免费、速度快、支持自定义域名）
