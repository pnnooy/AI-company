# 桌面学术助手机器人 - 核心项目规范书 (V2.1)

## 一、 项目概述
本项目为一款面向大学生的桌面级学术助手机器人。系统已全面从标准库（SPL）迁移至 **STM32CubeMX + HAL库** 的现代工业级开发架构。设备旨在通过视觉（LCD表情）、氛围光（RGB灯）、触觉（电容触摸）、姿态（六轴传感器）以及实体道具（NFC卡片）实现多模态的桌面交互。

---

## 二、 核心硬件拓扑与引脚映射
当前系统移除了早期验证阶段的冗余传感器，专注于以下核心交互模块。引脚分配已针对 STM32F103VET6 的原生硬件资源进行了最优映射。

| 模块分类 | 硬件设备 | 核心交互功能 | 接口协议 | STM32 物理引脚映射 |
| :--- | :--- | :--- | :--- | :--- |
| **中枢通信** | CH340 (USB转TTL) | 接收 PC 端 AI 决策指令，上报传感器事件 | **USART1** | `PA9` (TX), `PA10` (RX) |
| **视觉呈现** | 3.2寸 ILI9341 屏幕 | 承载机器人的动态面部表情系统 | **FSMC** | 板载 8080 并口插槽 |
| **氛围指示** | RGB 全彩 LED | 配合情绪输出环境光效 (如专注蓝光) | **TIM3 (PWM)** | `PB5` (红), `PB0` (绿), `PB1` (蓝) |
| **触觉感知** | TTP223 触摸模块 A | 贴于左侧外壳，感知用户“抚摸”等动作 | **EXTI** (中断) | `PC4` (上升沿触发) |
| **触觉感知** | TTP223 触摸模块 B | 贴于右侧外壳，提供组合触控输入 | **EXTI** (中断) | `PC5` (上升沿触发) |
| **姿态前庭** | MPU6050 六轴 IMU | 感知设备跌倒、拿起或剧烈摇晃 | **I2C1** (硬件) | `PB6` (SCL), `PB7` (SDA) |
| **实体交互** | MFRC-522 读卡器 | 识别不同 NFC 实体卡片触发特定模式 | **SPI1** (硬件) | `PA5` (SCK), `PA6` (MISO), `PA7` (MOSI), `PA4` (CS) <br> `PA3` (RST) |

## 三、 CubeMX 底层配置基准
后续 Agent 在接手或重新生成工程时，必须严格遵守以下底层配置基准：

* **MCU 选型**：STM32F103VET6。
* **时钟树 (RCC)**：启用外部高速晶振 (HSE)，配置系统主频至最高 (72MHz)。
* **调试接口 (SYS)**：必须配置为 **Serial Wire (SWD)**，防止下载后锁死芯片。
* **波特率**：USART1 统一配置为 **115200 bps**，参数为 8-N-1。
* **中断优先级**：EXTI 外部中断的优先级需低于系统核心滴答定时器（SysTick），以免阻塞主循环。

## 四、 软件系统状态与重构任务
基础的 HAL 库工程框架已搭建完毕，以下是后续接手 Agent 需要重点开发的业务逻辑：

### 1. 表情视觉引擎 (待重构)
* **状态说明**：早期基于 C89 标准库的像素画法已废弃。当前仅保留表情枚举定义（如 `EMO_NORMAL`, `EMO_HAPPY`, `EMO_FOCUS`, `EMO_ANGRY` 等）。
* **接手任务**：需设计并实现一套全新的、基于 HAL 库的动态表情系统。具体实现方式由接手 Agent 全权设计。

### 2. 硬件抽象驱动挂载 (待编写)
* **任务 A**：编写/移植基于硬件 SPI1 的 RC522 HAL 库底层读写操作，实现对 S50 白卡的 UID 获取。
* **任务 B**：编写/移植基于硬件 I2C1 的 MPU6050 姿态解算代码，通过读取原始加速度与角速度，输出简化的姿态事件（如“倾倒”或“平稳”）。

### 3. 主循环调度 (待开发)
* **状态说明**：废弃所有 `Soft_Delay` 死延时。
* **接手任务**：在 `main.c` 中利用 `HAL_GetTick()` 构建非阻塞式的状态机框架。以极短的周期刷新屏幕（驱动新表情系统），同时异步轮询触摸事件和 NFC 刷卡事件。

## 五、 开发环境与工具链
* **图形化配置**：STM32CubeMX (生成初始化代码)。
* **跨平台编译**：支持在 Windows 11 本地生成 Keil MDK-ARM (V5) 工程进行调试。
* **代码规范**：全面启用现代 C 语言标准（C99及以上），利用 HAL 库的面向对象特性进行封装。
