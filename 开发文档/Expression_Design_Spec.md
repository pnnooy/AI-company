## 一、技术硬约束（必须遵守）

| 参数 | 值 | 为什么 |
|------|-----|--------|
| 画布尺寸 | **80 × 80 像素** | MCU 内存有限，2x 缩放后屏显 160×160 |
| 背景色 | **纯黑 #000000** | LCD 通过透明叠加实现表情切换，黑=透明 |
| 颜色数量 | **无硬限制**，但建议 ≤32 色 | 少颜色 = 方便修改 + 像素风更干净 |
| 导出格式 | **PNG-24** (RGB, 不带透明通道) | 转换脚本只读 RGB 三个通道 |
| 文件名 | **严格按上表**，不得改 | 脚本靠文件名识别表情类型和帧号 |

### ⚠️ 背景必须是纯黑的

这是最重要的技术约束。LCD 上表情覆盖在黑色背景上，所有 `#000000` 的像素 = 不显示（露出底层黑色），所有非黑像素 = 显示。

**如果你不确定颜色是否纯黑，在 PS 里用油漆桶填 `#000000` 填满背景层。**

### ⚠️ 关于白色

LCD 的白色是 `#FFFFFF` 转成 RGB565 后的 `0xFFFF`。白色线条在黑色背景上显示效果很好。但注意**不要用纯白做大面积填充**，会显得刺眼。建议白色只用于描边和高光。

## 二、设计风格指南

### 总体风格：圆润可爱像素风

```
参考对象:   Line Friends / Kakao Friends / 聊天表情
核心原则:   少即是多 — 80×80 画布很小，细节太多反而糊
线条:      2px 白色或浅色描边，不用黑色描边（黑底上黑线看不见）
填充:      明亮、饱和度高的颜色
眼睛:      黑色圆形，直径约 8-14px，加白色高光点
嘴巴:      2-3px 线条
```

### 颜色调色板建议

```
皮肤/脸:   #FFD89B (暖黄)   #FFE0B2 (浅暖黄)   #FFCC80 (橘调)
腮红:     #FF9E9E (粉)     #FFAB91 (橘粉)
白色:     #FFFFFF (纯白, 描边/高光)
眼睛:     #1A1A2E (深黑蓝)  #000000 (纯黑)
嘴/线条:  #4A4A4A (深灰)    #FFFFFF (白线)
红色强调: #FF5252 (生气/爱心)
蓝色强调: #448AFF (专注相关)
```

### 脸部构图参考（80×80 画布内）

```
       0   10   20   30   40   50   60   70  80
   0  ┌────────────────────────────────────────┐
      │              (头顶留白 2px)             │
  10  │         ┌──────────────────┐           │
      │         │    脸部圆形       │           │
  20  │         │   直径 ~70px     │           │
      │    ● (左眼)       ● (右眼)  │           │
  30  │   x≈28,y≈32      x≈52,y≈32 │           │
      │   直径 10-12px              │           │
  40  │                             │           │
      │       ∩  (嘴巴, y≈52)      │           │
  50  │                             │           │
      │         └──────────────────┘           │
  70  │              (下巴留白 2px)             │
  80  └────────────────────────────────────────┘
```

数字参考：
- 脸部圆形：圆心 (40,40)，半径 35-38px
- 左眼圆心：~(28, 32)，右眼圆心：~(52, 32)
- 眼睛直径：正常 10-12px，惊讶 14-16px，闭眼时 2px 横线
- 嘴巴中心 y 坐标：~52-55
- 嘴巴宽度：正常 16-20px，大笑 24-30px

## 三、每种表情的详细设计要求

### 1. 普通/默认 `emo_normal` — 1 帧

```
文件名:    emo_normal_f0.png
帧数:      1 (静态)
触发场景:  开机默认、无交互待机、PC 未连接
```

**视觉效果**：
- 圆形脸部填充暖黄色 `#FFD89B`
- 白色描边 2px（脸部外轮廓）
- 黑色圆眼，直径 ~10px，眼内各一个白色高光点（2×2px，位置偏右上）
- 嘴巴：一条平直线或微弯弧线，2px 粗，深灰色
- 可选：两个小腮红圆（淡粉 `#FF9E9E`，直径 6-8px），位于眼下外侧

**情绪传达**：中性、平静、等待中。

---

### 2. 开心 `emo_happy` — 3 帧（眨眼动画）

```
文件名:    emo_happy_f0.png  (睁眼笑)
          emo_happy_f1.png  (眨眼-半闭)
          emo_happy_f2.png  (睁眼笑, 与 f0 相同可复用)
动画节奏:  每 200ms 切换一帧, f0→f1→f2→f0 循环
触发场景:  用户触摸、收到夸奖、交互响应
```

**f0 (睁眼笑)**：
- 基础脸型同 NORMAL
- 眼睛变成上弯弧线 `^  ^`（用黑色 2-3px 弧线替代圆形眼睛）
- 嘴巴：上弯大笑弧线，宽度 ~24px，嘴内可填深红色做张嘴笑
- 可选：眼角外侧加小弧线强调笑意

**f1 (半闭眼)**：
- 与 f0 唯一区别：眼睛从上弯弧线变为半闭状态
- 半闭眼画法：黑色半椭圆（上半保留，下半拉平），或直接画成扁平横线 + 上方的上弯弧线
- 嘴巴保持不变

**f2 (睁眼笑)**：
- 与 f0 完全相同（直接复制 f0.png 即可）

> **💡 技巧**：f2 可以直接用 f0 的 PNG，因为动画循环是 f0→f1→f0→f1，所以可以省略 f2。
> 但如果想循环做 f0→f1→f2→f0（两帧睁眼夹一帧眨眼），保留 f2 即可。

**情绪传达**：开心、欢迎、活泼。

### 3. 专注 `emo_focus` — 2 帧

```
文件名:    emo_focus_f0.png
          emo_focus_f1.png
动画节奏:  每 500ms 交替
触发场景:  NFC 学习卡刷卡、用户问学术问题、长时间注视
```

**f0**：
- 基础脸型同 NORMAL
- 眼睛：略微眯起，黑色椭圆（宽 > 高），或正常圆眼但瞳孔缩小
- 可加眼镜框：深蓝 `#448AFF` 或白色方框眼镜，2px 线宽
- 嘴巴：紧闭直线或微微下压
- 可选：额头加一两道竖线（专注纹）

**f1**：
- 与 f0 的区别：眼镜框上的高光位置微移（模拟镜片反光变化）
- 或者眼睛瞳孔位置微移
- 变化要**非常细微**，营造"认真盯着看"的感觉而不是大幅度动画

> **💡 如果不会画眼镜，可以省略眼镜**。纯靠眯眼 + 紧嘴来表达专注，2 帧通过瞳孔大小微调实现。

**情绪传达**：专注、认真、思考中。

### 4. 生气 `emo_angry` — 1 帧

```
文件名:    emo_angry_f0.png
帧数:      1 (静态)
触发场景:  设备倾倒告警、摇晃过度、异常状态
```

**视觉效果**：
- 基础脸型：可以用比平时稍深一点的肤色（偏红）
- 眉毛：**关键元素** — 倒八字眉 `\  /`，用深红色 `#FF5252` 或黑色画 2px 粗线，倾斜约 30°
- 眼睛：锐利的三角眼或小圆眼，黑色填充，白色高光极小（或不要高光）
- 嘴巴：下弯弧线（⇂），2-3px 粗，深红或黑色
- 可选：额头加红色愤怒符号 `💢`（但 80×80 画布可能太小）

**情绪传达**：愤怒、警告、异常。

### 5. 困倦/休眠 `emo_sleep` — 2 帧

```
文件名:    emo_sleep_f0.png  (闭眼)
          emo_sleep_f1.png  (微睁眼)
动画节奏:  每 1000ms 交替 (慢速, 模仿呼吸节奏)
触发场景:  长时间无人交互进入休眠、深夜时段
```

**f0 (闭眼)**：
- 基础脸型：肤色可以偏冷/偏暗一点
- 眼睛：两条水平横线 `—  —`（闭眼线），2px 粗，深灰色
- 嘴巴：小圆圈或小椭圆（打呼嘴型），直径约 6-8px
- 可选：鼻孔上方飘出 "Z" 或 "💤" 符号（但 80×80 画布很挤，建议用 1 个 Z）

**f1 (微睁眼)**：
- 与 f0 的区别：闭眼线变成半睁状态（黑色线下方加一点点瞳孔）
- 或者眼睛保持横线但位置微微下移（放松感）
- 嘴型保持不变

**情绪传达**：困倦、休眠、请勿打扰。

```
文件名:    emo_sad_f0.png  (有泪滴)
```

- 眉毛和嘴保持不变

**情绪传达**：喜爱、心动、被宠爱。

## 四、工作流程

### Step 1: 确认环境

```bash
# 安装 Pillow (PNG 处理库)
pip install Pillow

# 确认脚本可用
python tools/png2rgb565.py
# 应输出帮助信息
```

### Step 2: 开始画

1. 新建 80×80 画布，背景填 `#000000`
2. 先画 `emo_normal_f0.png`（这是基准脸，其他表情在它基础上改）
3. 画完一张就导出一张 PNG-24 到 `firmware/Assets/` 目录
4. 建议按这个顺序画：NORMAL → HAPPY → SAD → ANGRY → SURPRISE → FOCUS → SLEEP → LOVE

### Step 3: 转换验证

```bash
# 单张转换
python tools/png2rgb565.py firmware/Assets/emo_normal_f0.png

# 全部转换
python tools/png2rgb565.py firmware/Assets/emo_*.png

# 验证生成的 C 数组
python tools/png2rgb565.py --check firmware/App/emo_normal_f0.c
# 应输出: OK: ... 80×80, 6400 pixels, range [0x0000, 0xXXXX]
```

### Step 4: 更新固件

```bash
# 一次性生成 expression_assets.c/.h
python tools/make_assets.py --size 80

# 然后在 Keil 中 F7 编译, F8 烧录
```

### Step 5: 在板子上验证

- 用 SSCOM 连接 COM6，发命令切换表情看效果
- 检查颜色是否正确（红色和蓝色有没有互换 — 之前出现过这个 bug）
- 检查动画帧切换是否流畅

## 五、常见问题 & 陷阱

### Q: 我画了浅灰色背景而不是纯黑，有关系吗？
**A**: 有！LCD 会把所有非黑像素显示出来，灰色背景会显示成灰色方块。**背景必须是 #000000**。

### Q: 80×80 太小了，我能画 160×160 然后缩吗？
**A**: 可以，但建议直接用 80×80 画，因为缩小时线条会变模糊。如果一定要画大图，缩小时用**最近邻（Nearest Neighbor）**插值，不要用双线性。

### Q: 能加渐变/半透明吗？
**A**: 不能。RGB565 不支持透明度，所有半透明导出后会变成不透明的实色。建议纯色块 + 硬边缘。

### Q: 我画的 PNG 在电脑上看很漂亮，烧到板子上颜色不对？
**A**: 这是已知的 LCD 颜色字节序问题。如果颜色明显不对（比如脸红变蓝），告诉 B 角色调整 ILI9341 的颜色映射，不需要你改图。

### Q: 需要把脸画满 80×80 吗？
**A**: 不需要。脸部圆形直径建议 ~70px，四边各留约 5px 黑边。这样在 160×160 的缩放显示中，脸周围有 10px 黑边，看起来舒服。
