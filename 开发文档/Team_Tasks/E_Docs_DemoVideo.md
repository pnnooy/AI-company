### 3.3 技术方案

纯前端，一个 HTML 文件，内嵌 CSS + JS，不依赖任何框架。

用到的技术：
- **CSS `@keyframes`**：表情切换动画、呼吸灯效果
- **CSS `scroll-snap`** 或简单的滚动触发：翻页效果
- **JS IntersectionObserver**：滚动到某个区域时触发动画
- **JS Canvas** 或纯 CSS 画机器人脸：不需要图片也能演示

### 3.4 核心动画：CSS 表情切换演示

```css
/* 机器人脸 — 纯 CSS 实现 */
.robot-face {
    width: 160px; height: 160px;
    background: #000;
    border-radius: 50%;
    position: relative;
    /* 脸部圆形 */
}
.robot-face .face-circle {
    width: 140px; height: 140px;
    background: #FFD89B;
    border-radius: 50%;
    border: 3px solid #fff;
    position: absolute;
    top: 10px; left: 10px;
}
/* 眼睛 — 两个黑色圆 */
.eye {
    width: 20px; height: 20px;
    background: #000;
    border-radius: 50%;
    position: absolute;
    top: 55px;
}
.eye.left { left: 45px; }
.eye.right { right: 45px; }

/* 开心表情：眼睛变弯线 */
.face-happy .eye {
    width: 24px; height: 12px;
    border-radius: 0 0 24px 24px;  /* 下半圆 = 弯眼 */
    background: #000;
}
/* 生气表情：加眉毛 */
.face-angry .eyebrow {
    width: 28px; height: 4px;
    background: #FF5252;
    position: absolute;
    top: 42px;
    transform: rotate(-25deg);
}
.face-angry .eyebrow.right {
    transform: rotate(25deg);
    right: 38px;
}
```

### 3.5 交互演示区实现思路

```
页面中间一个机器人脸部动画区，旁边是触发按钮：

  ┌────────────────────┐
  │                    │    [🙂 摸左边]
  │   (CSS 动画脸)     │    [💳 刷NFC卡]
  │                    │    [📳 摇晃]
  │                    │    [⚠️ 倾倒]
  └────────────────────┘    [⏱ 长时间不动]

用户点击按钮 → JS 切换 face 的 class → CSS transition 过渡到新表情
```

### 3.6 技术架构图动画

用简单 CSS 动画展示数据流：

```
[STM32] --串口--> [Python AI Engine] --HTTP--> [Web UI]
   ↑                  ↓

   ↑                  ↓
[触摸/NFC/IMU]    [LCD表情/RGB灯]
```

每点击一个交互按钮，流程线上高亮对应的数据流向。

### 6.1 开发日志

在整个开发周期中，每天记录：

```markdown
# 开发日志 2026-06-05

## 今日进展
- [A] NFC 读到 UID，VersionReg 返回 0x92 正常
- [B] 串口协议对通，收到第一个触摸事件帧
- [C] Web 面板完成，mock 数据调试通过
- [D] 完成 4 种表情线稿
- [E] HTML 讲解页框架完成

## 遇到的问题
- SPI 时钟需要降到 4.5MHz 才稳定
- 协议 CRC-8 计算差一个字节偏移

## 解决方案
- CubeMX 改 SPI 分频
- 统一 CRC 计算范围：从 CMD 开始算
```
