# 角色 E：项目报告撰写

> **负责人**: E
> **工期**: 2-3 天（贯穿开发周期）
> **前置条件**: 老师提供的报告格式要求、各组的产出物

---

## 一、任务总览

你的核心任务是**按照老师的要求撰写项目报告初版**。

核心原则：
- **格式为王**：把报告框架、目录结构、排版、封面等格式打磨完美
- **内容让 AI 填**：正文内容使用 AI（如 Claude、ChatGPT）生成填充，不需要你自己从头写
- **框架是交付物**：一个好的模板 + 完善的目录 + 正确的格式 = 及格起步

| 阶段 | 任务 | 时间 |
|------|------|------|
| 第一天 | 搞清楚格式要求 + 搭建报告框架 | 2-3h |
| 第一天 | 把项目背景资料喂给 AI，生成各章节初稿 | 2h |
| 第二天 | 补充各组的最新进展，调整格式细节 | 2-3h |
| 第三天 | 最终排版 + 目录 + 封面 + 交付 | 2h |

---

## 四、第二步：搭建报告框架

### 4.1 建议的章节结构（通用学术报告格式）

```
封面
  - 项目名称：桌面学术助手机器人
  - 组员姓名 + 学号
  - 指导教师
  - 日期

摘要
  - 中文摘要（200-300字）
  - 关键词（3-5个）

目录
  - 自动生成的目录

第一章  项目背景与意义
  1.1 项目背景
  1.2 项目目标
  1.3 国内外研究现状
  1.4 项目创新点

第二章  系统总体设计
  2.1 系统架构概述
  2.2 硬件拓扑设计
  2.3 软件架构设计
  2.4 通信协议设计

第三章  硬件模块设计与实现
  3.1 STM32 主控模块
  3.2 LCD 显示模块（ILI9341）
  3.3 RGB LED 氛围灯模块
  3.4 触摸传感器模块（TTP223）
  3.5 姿态传感器模块（MPU6050）
  3.6 NFC 读卡模块（MFRC-522）
  3.7 串口通信模块（CH340）

第四章  软件系统设计与实现
  4.1 固件层设计（STM32 HAL）
  4.2 PC 后端设计（Python）
  4.3 AI 决策引擎
  4.4 Web 控制面板

第五章  系统测试与分析
  5.1 硬件模块测试
  5.2 通信链路测试
  5.3 系统集成测试
  5.4 测试结果分析

第六章  总结与展望
  6.1 项目总结
  6.2 存在的问题
  6.3 未来改进方向

参考文献

致谢

附录
  附录A 完整代码仓库地址
  附录B 硬件接线图
  附录C 通信协议规格表
```

### 4.2 格式框架搭建（在 Word 中）

1. **新建文档** → 设置页面：A4，页边距上下 2.54cm，左右 3.18cm（或按老师要求）
2. **定义样式**：
   - 标题 1：黑体/三号/加粗/居中
   - 标题 2：黑体/四号/加粗/左对齐
   - 标题 3：黑体/小四/加粗/左对齐
   - 正文：宋体/小四/1.5 倍行距/首行缩进 2 字符
   - 代码块：Consolas/五号/单倍行距/灰色底纹
3. **插入目录**：用 Word 自动目录功能（基于标题样式）
4. **插入页码**：封面不标，正文从第 1 页开始
5. **设置页眉**：奇数页写"桌面学术助手机器人"，偶数页写章节标题

**这一步做完，你就已经有了一个 70 分的框架**。剩下的内容是填充，交给 AI 就行。

## 五、第三步：用 AI 填充报告内容

### 5.1 喂给 AI 的素材

把以下素材整合成一段 prompt，发给 AI（Claude / ChatGPT 等）：

```
你要写的项目信息：
1. 项目名称：桌面学术助手机器人 (Desktop Academic Assistant Robot)
2. 平台：STM32F103VET6 + PC (Python)
3. 硬件模块：ILI9341 LCD、RGB LED (TIM3 PWM)、TTP223 触摸、
   MPU6050 六轴传感器、MFRC-522 NFC 读卡器、CH340 串口
4. 软件：固件用 HAL 库 + FreeRTOS，PC 用 Python (pyserial + FastAPI)
5. AI 引擎：状态机 + 规则引擎，后续接入 LLM
6. 通信：USART1 115200bps，自定义二进制帧协议 (CRC-8 校验)
7. 功能：
   - 8 种动态表情显示（正常、开心、专注、生气、休眠、惊讶、难过、爱心）
   - RGB 全彩氛围灯随情绪变化
   - 触摸交互（单击/双击/长按）
   - NFC 刷卡识别
   - 姿态检测（摇晃/倾倒）
   - PC 端 AI 实时决策
   - Web 控制面板

项目背景：面向大学生的桌面级学术助手，通过多模态交互缓解学习压力、
提升学习效率。

详细资料见项目文档：
- 开发文档/Project_Brief.md（项目规范）
- 开发文档/Development_Plan.md（开发计划）
- CLAUDE.md（项目总规范）
```

### 5.2 逐章让 AI 生成

**不要一次让 AI 写整份报告**，AI 会偷懒。逐章提问：

**第 1 章提示词**：
```
你是电子信息工程专业的大三学生。请为《桌面学术助手机器人》项目报告写第一章"项目背景与意义"，包括背景、目标、研究现状、创新点。学术风格，1500字左右。不要虚构参考文献。
```

**第 2 章提示词**：
```
请写第二章"系统总体设计"。包括系统架构概述（配文字描述，不用画图）、硬件拓扑、软件架构、通信协议。描述要具体，提到 STM32F103VET6、ILI9341 LCD、MPU6050、RC522 等具体模块。1500字。
```

**依此类推**，每章单独提问，把本章相关的技术细节附在提示词里。

### 5.3 AI 生成后的处理

AI 生成的内容**不能直接用**，你需要：

1. **检查事实错误**：AI 可能会编造引脚号、协议细节等。对照项目文档修正
2. **统一术语**：确保全文用词一致（比如不要一会儿"微控制器"一会儿"单片机"）
3. **调整语气**：确保符合学术报告风格（不要有 AI 味太重的话如"在当今时代..."）
4. **删除废话**：AI 喜欢写套话，果断删
5. **补充真实数据**：让各组提供实测数据（NFC 读取成功率、触摸响应延迟等）
